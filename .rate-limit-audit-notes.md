## Issues found (summary → see issue body for detail)
1. relay/rate-limiter.ts: stateless sleep on EVERY call (incl. first) — over-throttles cold-start.
2. relay/server.ts JSDoc describes "time since last request" semantics the code does NOT implement (mixup with Throttle).
3. relay rate-limiter correctness relies on max_concurrent:1 per profile, but max_concurrent is OPTIONAL and defaults to global queue.concurrency.
4. Double throttle when relay enabled: in-process Throttle still installed; no guard against relay.
5. Throttle is per-job (Throttle instance created inside runJob) — no cross-job coordination when relay is OFF and concurrency>1.
6. server.ts JSDoc "no retries / no Retry-After" contradicts forwarder.ts which implements both.
7. Default retry_base_delay_ms=3000 vs api_rpm=30 interval=2000ms violates schema's own "≥2x interval" rule.
8. README framing "throttle floor" inconsistent with schema "caps RPM" / intent.
